## 2026-07-18 02:14 — Config schema migration + identity governance fix

**事件：** openclaw.json agents 配置格式迁移

**根因：** 原始配置用 `agents.main` / `agents.tg-agent`（对象属性 key），但 OpenClaw 2026.5.20 schema 要求 `agents.list[]`（数组 + id 字段）。触发 `agents: Invalid input` 校验失败。

**触发链：** `openclaw gateway restart` → config invalid → `openclaw doctor --fix` → 从 last-known-good 恢复（仅 defaults，丢失 agents.main + tg-agent + bindings）

**修复：** 手动写入 schema 兼容格式：

```json
agents:
  defaults: { ... }
  list:
    - { id: "main",     identity: { name: "燃🔥", emoji: "🔥" } }
    - { id: "tg-agent", identity: { name: "烬🔥", emoji: "🔥" }, workspace: "..." }
```

bindings（顶层数组）：
```json
bindings:
  - { agentId: "tg-agent", match: { channel: "telegram" } }
```

**治理经验（重要）：**
- `doctor --fix` 恢复的是"默认可运行状态"，不是"当前架构状态"
- 以后在有多 agent / identity governance 的系统上：
  1. 优先检查 schema 兼容性
  2. 手动迁移而非依赖 doctor
  3. 备份后再改
- schema migration 必须先于 identity migration
- 配置格式 ≠ 治理模型

**标记：**
- Telegram 的 `accounts.default` 缺失警告未处理（不影响运行，`defaultAccount: "main"` 回退启用）
- Secret 明文迁移延期
