## Abstraction Stability Principle — Governance Decision

**时间:** 2026-07-21 13:02
**来源:** 龙哥治理反馈 (webchat)
**类型:** architecture_decision
**涉及:** OBSERVER-ABSTRACTION-LAYER-v1.0.md

### 决策内容

抽象粒度应属于最慢演化层，增加 Abstraction Stability Principle：

1. **演化速度分层**：运行参数(日/小时) → Prompt(天) → 观察规则(周/月) → 抽象粒度(月/季度) → 核心契约原则(极慢)
2. **稳定层与变化层分离**：
   - 稳定层：等级定义、分类原则、判断边界
   - 变化层：事件映射、候选缺口、分析报告
3. **Abstraction Stability Principle**：多案例证据 / 现有等级无法表达 / 人工评审 / 不因单次事件调整
4. **抽象内容 vs 抽象实例**：内容稳定，实例变化

### 文档变更

- OBSERVER-ABSTRACTION-LAYER-v1.0.md: 新增 §3 演化速度与稳定性 + §8 抽象稳定性原则
- 全文重新编号 §1-§12
- Phase 1 加入"抽象粒度稳定性"为核心观察项

### 约束

- Phase 1 观察期内不新增抽象等级
- 不因单次事件修改等级定义
- 违反原则触发 Observer 重新校准

## 核查发现

```

### 实际执行环境

```
TG agent 运行时上下文可见:
  我是烬🔥
  + 可以写 memory

TG agent 运行时上下文不可见:
  烬🔥的数据必须写入 memory/events/jin/

### 不是的问题

| 排除项 | 原因 |
|:-------|:------|
| writer bypass | writer 行为正确，只是路由信息缺失 |
| 契约冲突 | 三份契约不冲突，各自管不同层面 |
| 需要删除契约 | 缺接入链而非契约内容错误 |
| 需要 runtime 守卫 | 软约束阶段，先验证接入是否足够 |
