## 19:08 — timer 维护排查经验

### timer 未触发的正确排查链

```
用户发现定时任务没执行
        ↓
1. timer 是否存在？         systemctl list-unit-files xxx.timer
2. timer 是否 active？      systemctl is-active xxx.timer
3. 是否在 timers.target？   ls timers.target.wants/xxx.timer
4. 到点是否触发？          journalctl -u xxx.timer
5. service 是否执行？       journalctl -u xxx.service
6. 脚本是否成功？           journalctl -u xxx.service 看 exit code
7. 链路是否完整？           查看下游日志（推送/事件等）
```

### 修改 timer 后的固定操作

```bash
systemctl --user daemon-reload
systemctl --user enable xxx.timer        # 重建 symlink
systemctl --user restart xxx.timer
systemctl --user status xxx.timer
systemctl --user list-timers xxx.timer   # 验证出现在排班中
```

修改 timer 文件后 `daemon-reload` 会导致 enable 状态丢失（symlink 被移除），必须显式重新 enable。多个 timer 同时修改时不要漏。

### 排查铁律

- 没有证据，不要给根因
- 最多说：观察到现象 X，可能原因包括 A/B/C，需要检查证据
- 被质疑时，回到证据查 journal，而不是继续解释
