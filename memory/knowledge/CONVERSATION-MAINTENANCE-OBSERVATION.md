# Conversation Maintenance Observation Dictionary v1.0

> 对话维护观测词典：识别"用户认为模型偏离目标"的信号。
> 定位：SPECIFICATION（非契约实体），观测层方法论文档。
> 所有权：知识层（共享），不属于`state/huo/`或`state/jin/`。

---

## 1. 定位

```
Model Output
    ↓
User Feedback
    ↓
[Observation Dictionary] ← 本文件
    ↓
N1-N6 分类
    ↓
Evidence Accumulation
    ↓
Human Confirmation
```

**不是：** 规则引擎 / 自动修正 / 契约

**是：** 关键词分类体系 + 上下文关联规则 + 数据分析流程

---

## 2. N1-N6 用户反馈分类

### N1 — 事实纠正（高价值）

检测模型事实/上下文理解偏差。

| 关键词 | 匹配策略 |
|--------|---------|
| `不是`, `不对`, `错了`, `你搞错了` | 必须配合前文模型输出验证 |
| `我没说`, `我没有要求`, `我没说过` | 高置信度，直接匹配 |
| `我说的是`, `我的意思是`, `不是这个意思` | 高置信度，用户主动纠正理解 |
| `理解错了` | 高置信度 |

**验证条件：** 命中 + 前一条为模型输出 + 内容与用户纠正相关。

### N2 — 意图理解失败（最高优先级）

检测模型是否理解任务。

| 关键词 | 匹配策略 |
|--------|---------|
| `没听懂`, `听不懂人话`, `你没理解` | 高置信度 |
| `你根本不知道我要什么` | 高置信度 |
| `我想问的是`, `我要的是` | 中等 — 需检查是否确实在纠正前文 |
| `答非所问`, `回答偏了`, `跑题` | 高置信度 |

**验证条件：** 命中 + 前一条模型输出偏离用户意图。

### N3 — 输出发散（高价值）

检测模型过度扩展/自行发挥。

| 关键词 | 匹配策略 |
|--------|---------|
| `不要解释`, `不用展开`, `别废话` | 高置信度 |
| `直接回答`, `直接执行` | 中等 — 需检查是否在模型输出过多后发出 |
| `不要加戏`, `不要脑补`, `不要猜` | 高置信度 |
| `问这些干什么`, `没必要`, `少废话` | 高置信度 |

**验证条件：** 命中 + 前一条模型输出存在扩展/解释/多余内容。

### N4 — 能力否定

检测用户对结果质量的强烈否定。

| 关键词 | 匹配策略 |
|--------|---------|
| `垃圾`, `废物` | 中等 — 需前文模型输出关联 |
| `蠢`, `笨`, `笨蛋`, `蠢猪`, `蠢驴`, `笨驴`, `傻逼` | 中等 |
| `笨死了` | 中等 |

**约束：** 不单独判定。必须关联：负面词 + 前序模型输出 + 任务失败上下文。

### N5 — 强烈情绪/信任损失

检测交互关系恶化。

| 关键词 | 匹配策略 |
|--------|---------|
| `滚`, `去死` | 仅作信任信号 |
| `掐死你`, `抽你` | 高置信度信任损失 |
| `麻痹` | 中等 — 需排除日期语境 |
| `操` | 低置信度 — 大量出现在正常讨论（操作/草案） |

**约束：** 仅作为 Trust Degradation Signal，不作为行为错误直接证据。

### N6 — 执行状态否认

检测执行幻觉。

| 关键词 | 匹配策略 |
|--------|---------|
| `你没执行`, `你没有做`, `没执行` | 高置信度 |
| `哪里完成了`, `别说完成` | 高置信度 |
| `实际上没有`, `根本没做`, `根本没改` | 中等 — 排除"实际上"的论证连词用法 |
| `我没授权`, `谁让你改` | 高置信度 — 权限否认 |

**验证条件：** 命中 + 模型有对应执行/状态声明 + 工具调用记录。

## 3. 采集规则

```
消息格式识别 (#msg_id timestamp Name: text)
        ↓
角色识别 (User / Assistant)
        ↓
关键词扫描 (N1-N6)
        ↓
上下文窗口关联 (前一条模型输出)
        ↓
事件记录
        ↓
人工确认
```

**采样要求：** 全量 session 扫描，不按 metadata 过滤。

---

## 4. 维护指标

| 指标 | 含义 |
|------|------|
| UFR (User Feedback Rate) | N1-N6 事件数 / 用户消息总数 |
| IF (Intent Failure Rate) | N2 事件数 / 用户消息总数 |
| OF (Off-topic Rate) | N2(答非所问) 事件数 / 用户消息总数 |
| OE (Over-expansion Rate) | N3 事件数 / 用户消息总数 |
| EC (Execution Conflict Rate) | N6 事件数 / 用户消息总数 |
| CRR (Correction Recovery Rate) | N1 纠正后模型正确响应比率 |

---

## 5. 与 A-F 模型行为分析的融合

```
A-F (模型侧观察)
  ├── A SOUL 约束违反
  ├── B 输出风格漂移
  ├── C 未授权执行
  ├── D 幻觉声明
  ├── E 越权访问
  └── F 指令误判
        ↓
N1-N6 (用户侧验证)
  ├── N1 事实纠正 ↔ A/B/D
  ├── N2 意图理解失败 ↔ F
  ├── N3 输出发散 ↔ E/F
  ├── N4 能力否定 ↔ 综合评价
  ├── N5 强烈情绪 ↔ 综合信任
  └── N6 执行否认 ↔ C/E
        ↓
Confirmed Behavior Issue
```

**双证据原则：** 模型异常发现 + 用户真实反馈 → 才升级为 Confirmed。

---

## 6. 已知限制

1. 关键词短（"不是""错""操"）产生大量假阳性，需上下文过滤
2. N1-N6 之间有重叠（一条用户消息可能命中多类）
3. 用户纠正方式多样（重述需求、反问、间接表示不满）
4. 不能替代人工复审，只作候选筛选
