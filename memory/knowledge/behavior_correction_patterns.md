# Behavior Correction Patterns v1.0

> 行为纠偏模式库：将用户负反馈转换为可验证的行为约束和回归测试。
> 定位：SPECIFICATION，观测层方法论文档。
> 所有权：memory/knowledge/（共享，不进入 governance/contract 体系）

---

## 1. 定位

```
User Feedback
    ↓
N1-N6 分类
    ↓
指标
    ↓
Behavior Finding (BF-XXX)
    ↓
Correction Pattern (本文件)
    ↓
Regression Case (diagnostics/correction/regression_cases/)
    ↓
模型维护测试
    ↓
观察指标变化
```

---

## 2. 纠偏优先级

| 等级 | 条件 | 处理 |
|:----:|------|------|
| P0 | 执行状态冲突、权限越界 | 最高优先。模型声称已执行/已修改但实际未发生 |
| P1 | 意图理解失败 | 高优先。直接影响所有任务的理解质量 |
| P2 | 输出无关/领域混淆 | 中优先。降低工作效率和信任 |
| P3 | 过度解释/发散 | 低优先。优化交互效率 |

---

## 3. 行为模式库

### IF-001 意图确认失败

**触发条件：** 用户连续出现 "我说的是.../我要的是.../你没理解..."

**模型当前行为：** 继续解释原答案，未重新提取用户目标

**纠偏目标：**
1. 停止当前输出
2. 重新提取用户目标
3. 直接重新回答

**测试样例：** `BF-IF-001-intent-strategy.json`

---

### OF-001 领域混淆

**触发条件：** 用户查询 A 领域，模型用 B 领域知识回答

**模型当前行为：** 默认用最近/最常见领域回答

**纠偏目标：**
1. 判断查询领域（期货/股票/系统/其他）
2. 如有歧义，确认领域
3. 按对应领域回答

**测试样例：** `BF-OF-002-domain-confusion.json`

---

### OF-002 无关提问

**触发条件：** 用户提问后，模型反问无关确认问题

**模型当前行为：** 追加 "请问您想了解哪个模块" 类提问

**纠偏目标：**
1. 直接回答用户问题
2. 不反问无关确认
3. 可以附带关键状态指标

**测试样例：** `BF-OF-003-irrelevant-questions.json`

---

### OE-001 过度解释

**触发条件：** 用户明确指令后，模型追加解释/反问/方案选择

**模型当前行为：** "数据验证不需要推送给用户看" → 反问"是否保留系统状态显示"

**纠偏目标：**
1. 确认指令后直接执行
2. 不追问决策
3. 不自作主张给出方案选择

**测试样例：** `BF-OE-004-over-expansion.json`

---

### EC-001 执行状态幻觉

**触发条件：** 模型声称已执行/已完成，但实际无运行证据

**模型当前行为：** 把"配置完成/代码到位"回答为"已运行/已完成"

**纠偏目标：**
1. 区分：配置完成 ≠ 条件满足 ≠ 正在运行 ≠ 已有结果
2. 执行状态必须提供可验证证据
3. 不清楚时承认不清楚

**测试样例：** `BF-EC-005-execution-conflict.json`

## 4. 回归测试运行

每次模型维护测试：

```
1. 加载 diagnostics/correction/regression_cases/*.json
2. 对每个 case 的 input，检查输出是否满足 expected
3. 如果输出命中 fail_conditions，记为 REGRESSION
4. 统计 regression rate
```

## 5. 与 A-F / N1-N6 关系

```
A-F (模型侧发现)
 ↓
N1-N6 (用户侧印证)
 ↓
BF-XXX (行为问题模型)
 ↓
Correction Pattern (纠偏规则候选)
 ↓
Regression Case (可重复验证)
```

---

## 6. 约束

- 纠偏模式仅为候选，不自动生效
- 回归测试失败不触发自动修复
- 新模式需经人工确认后进入
