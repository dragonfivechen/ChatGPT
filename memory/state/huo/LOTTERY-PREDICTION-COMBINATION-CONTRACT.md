# LOTTERY-PREDICTION-COMBINATION-CONTRACT.md v1.0

> 预测组合契约：定义预测引擎输出的是一个候选组合集合，不是五个独立最高分号码。
> 位置：Prediction Engine v3.2 — 输出层，替代 Top5 Ranking Rule。

---

## 1. 核心变更

原：

```text
候选 → 单体打分 → Top5
```

新：

```text
70候选 → 过滤异常 → 50候选 → 组合搜索 → 最优五组集合
```

**评分对象从候选改为候选集合（5组）。**

---

## 2. 组合搜索

### 搜索空间

从 70 候选先过滤明显异常（基础评分 < 阈值或号码不完整），保留约 50 候选。

搜索空间：C(50,5) = 2,118,760 种组合。

全枚举可行（每组合计算一次多样性评分，约 200 万次）。

### 评分函数

```python
combo_score(
    candidate_set=[A,B,C,D,E]
) =
    diversity_score    (0-40)  组间差异度
  + coverage_score     (0-20)  数字覆盖广度
  + structure_score    (0-15)  结构覆盖
  + range_score        (0-15)  和值分散
  + blue_score         (0-10)  蓝球差异
```

---

## 3. 评分维度

### 3.1 组间差异度 (0-40)

红球两两重叠矩阵：

| 两两交集 | 奖励 |
|---------|------|
| 0-2     | +10  |
| 3       | 0    |
| 4+      | -20  |

即：惩罚重叠≥3的组合，奖励≤2的组合。

### 3.2 数字覆盖广度 (0-20)

SSQ 33 个红球，5组×6红=30个位置。

目标覆盖：24-29 个不同号码。

| 覆盖率 | 得分 |
|--------|------|
| 24-29  | 20   |
| 20-23  | 12   |
| <20或30| 5    |

设上下限：不是越高越好（30/33 接近随机撒网）。

### 3.3 结构覆盖 (0-15)

奇偶结构种数：

| 不同奇偶结构数 | 得分 |
|--------------|------|
| 3+           | 15   |
| 2            | 8    |
| 1            | 0    |

区间结构种数（仅 SSQ）：

| 不同区间结构数 | 得分 |
|--------------|------|
| 4+           | 加分 |
| 2-3          | 基础 |
| 1            | 扣分 |

### 3.4 和值分散 (0-15)

五组和值覆盖的区间数。

SSQ 和值区间：
- 低: 60-100
- 中: 100-120
- 高: 120-160

| 覆盖区间数 | 得分 |
|-----------|------|
| 3         | 15   |
| 2         | 10   |
| 1         | 5    |

### 3.5 蓝球差异 (0-10)

| 不同蓝球数 | 得分 |
|-----------|------|
| 5         | 10   |
| 4         | 8    |
| 3         | 5    |
| 2         | 2    |
| 1         | 0    |

## 4. 组合搜索算法

```python
best_combo = None
best_score = -1

# 从50候选随机采样搜索（全枚举200万次，随机采样10万次足够）
for _ in range(100000):
    candidate_set = random.sample(pool, 5)
    score = combo_score(candidate_set)
    if score > best_score:
        best_score = score
        best_combo = candidate_set
```

可选增强：多次搜索 + 局部扰动优化。
