# LOTTERY-PREDICTION-ROLE-CONTRACT.md v1.0

> 五组角色契约：定义最终输出的五组号码各自的角色定位。
> 位置：Prediction Engine v3.2 — 组合选择层，定义角色分类和分配规则。

---

## 1. 五组角色

```
最终5组:

G1 趋势延续组    保留近期频率特征
G2 热冷平衡组    热号+冷号混合  
G3 熵扰动组      基础候选变异
G4 反常识组      偏离历史结构
G5 防守覆盖组    最大结构覆盖
```

---

## 2. 角色定义

### G1 趋势延续组

| 属性 | 说明 |
|------|------|
| 策略来源 | trend_balance 池 |
| 选号倾向 | 近期高频号码为主 |
| 和值范围 | 中和值（均值±15） |
| 结构 | 2-2-2 或 3-2-1 |
| 评分权重 | 单组质量优先 |

### G2 热冷平衡组

| 属性 | 说明 |
|------|------|
| 策略来源 | hot_cold_mix 池 |
| 选号倾向 | 热号+温号+冷号混合 |
| 结构 | 均衡 2-2-2 |
| 评分权重 | 热冷比例合理性 |

### G3 熵扰动组

| 属性 | 说明 |
|------|------|
| 策略来源 | entropy 池 |
| 选号倾向 | 基础候选的变异体 |
| 特点 | 与 G1/G2 重叠最小 |
| 评分权重 | 与已选组差异度 |

### G4 反常识组

| 属性 | 说明 |
|------|------|
| 策略来源 | anti_pattern 池 |
| 选号倾向 | 低区偏重 / 高区偏重 / 连号结构 |
| 结构 | 偏态（1-1-4, 4-1-1, 0-3-3 等） |
| 评分权重 | 反常识得分 + 与已选组差异 |

### G5 防守覆盖组

| 属性 | 说明 |
|------|------|
| 策略来源 | 全策略池 |
| 选号倾向 | 最大化整体号码覆盖 |
| 目标 | 覆盖已选4组未覆盖的号码区域 |
| 评分权重 | 覆盖补全度 |

---

## 3. 选择算法

```
70候选

  ↓ 按策略分类

趋势池(10)  冷热池(10)  遗漏池(10)  约束池(10)  熵池(20)  反常识池(10)

  ↓ 各池内部按单组评分排序
  ↓ 保留 top 5

  ↓ 逐角色选择

G1: 趋势池中在趋势候选里选最高分
G2: 冷热池中选与G1差异最大的最高分
G3: 熵池中选与G1/G2差异最大的最高分
G4: 反常识池中选与已选差异最大的最高分
G5: 全池中选覆盖补全度最高的
```

---

## 4. 角色评分维度

| 角色 | 主维度 | 次维度 |
|------|--------|--------|
| G1 | 趋势吻合度（score） | 和值正常 |
| G2 | 热冷比例 2:2:2 | 与G1差异 |
| G3 | 与已选组差异度 | 变异质量 |
| G4 | 反常识得分 | 结构偏态度 |
| G5 | 覆盖补全度 | 与已选组差异 |

## 5. 差异度量

与已选组的差异度计算：

```python
def diversity(candidate, selected):
    nums_c = set(candidate_red)
    overlaps = [len(nums_c & set(s_red)) for s in selected]
    max_overlap = max(overlaps) if overlaps else 0
    return -max_overlap  # 重叠越小越好
```

## 7. 替代关系

本契约替代 `LOTTERY-PREDICTION-COMBINATION-CONTRACT.md`。

后者定义了组合搜索（任意5组采样），前者定义了角色化选择（每角色选最优1组）。
