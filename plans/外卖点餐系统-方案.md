# 外卖点餐系统 · 方案 v1.1

> 2026-07-17 | 状态：待评审 | 版本：v1.1
> 变更：技术方案从 H5 改为微信小程序

---

## 一、需求定位

微信小程序外卖点餐系统。顾客扫码打开小程序点餐，商家后台接单管理。

---

## 二、功能范围

### 顾客端（微信小程序）

| 模块 | 功能 |
|------|------|
| 菜单浏览 | 分类展示商品（名称/价格/图片/描述） |
| 购物车 | 加/减/清空/显示小计 |
| 下单 | 选择商品 → 确认信息 → 提交订单 |
| 订单查询 | 查看历史订单 + 当前订单状态 |
| 店铺信息 | 店名/公告/营业时间/联系方式 |
| 微信授权 | 获取用户昵称/头像（简化填写） |

### 商家端（Web后台）

| 模块 | 功能 |
|------|------|
| 商品管理 | 增/删/改/分类/上下架 |
| 订单管理 | 实时新订单提醒 / 接单 / 完成 / 历史查询 |
| 店铺设置 | 店名/营业时间/公告 |
| 数据概览 | 今日订单数/营收 |

### 第一版不包含

- 在线支付（先到付/微信转账，后续接入微信支付）
- 骑手配送跟踪
- 多店聚合
- 评价系统
- 会员/积分

## 三、技术选型

### 小程序端（三选一）

| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **A. 原生微信小程序** | 专注小程序 | 官方支持最全，性能最好 | 代码只能用在微信 |
| **B. Uni-app** | 可能跨多端 | 一套代码 多端编译 | 多了框架层 |
| **C. Taro** | React 生态 | React 语法 | 同上 |

**建议：选方案 A（原生）**，项目目标明确就是微信小程序，不用引入额外框架增加复杂度。微信官方工具 + 原生开发效率不低。

### 后端

| 项目 | 方案 A（自建服务器） | 方案 B（微信云开发） |
|------|-------------------|-------------------|
| 服务器 | 本机 Node.js | 微信云托管/云函数 |
| 数据库 | SQLite / PostgreSQL | 云数据库 |
| 成本 | 0（已有服务器） | 按量计费 |
| 运维 | 需自己管 | 托管 |
| 灵活性 | 高 | 受限于微信生态 |

**建议：方案 A（自建）**，服务器已有，零新增成本且灵活。后期量大了可直接切微信云托管。

### 具体技术栈

```
小程序端
  ├── 微信原生小程序
  ├── WeUI（微信官方UI库）
  └── 微信开发者工具开发

后端
  ├── Node.js v22 + Express
  ├── SQLite（单文件，日均几百单够用）
  ├── REST API
  └── PM2 守护进程

部署
  ├── 本机独立端口
  ├── 内网访问调试
  ├── 公网反代（需要时再加）
  └── 小程序后台配置合法域名
```

## 四、数据库设计

### 主要表

| 表名 | 说明 | 关键字段 |
|------|------|---------|
| shops | 店铺信息 | id, name, notice, open_time, close_time, phone |
| categories | 商品分类 | id, name, sort_order, shop_id |
| products | 商品 | id, name, price, description, image, category_id, status |
| orders | 订单 | id, shop_id, user_info, total, status, note, created_at |
| order_items | 订单明细 | id, order_id, product_id, quantity, price |

### 订单状态

```
待确认 → 已接单 → 制作中 → 配送中 → 已完成
                 ↘ 已取消
```

## 五、支付方案

| 方案 | 说明 | 门槛 | 建议 |
|------|------|------|------|
| A. 到付/转账 | 备注微信/支付宝，线下转账 | 无 | **第一版先用** |
| B. 微信支付商户号 | 小程序内直接支付 | 需企业/个体户营业执照 | 有证后接入 |
| C. 服务商模式 | 挂靠服务商 | 手续费 | 可选过渡 |

## 六、小程序上线流程（额外步骤）

```
注册微信小程序账号
  └── 个人主体（功能受限，无支付）
  └── 企业/个体户主体（完整功能）
        ├── 开发者资质认证（300元/年）
        └── 申请微信支付商户号

开发完成 → 提交审核 → 审核通过 → 上线发布
```

**你需要决定的：** 用个人账号还是企业主体？

## 七、开发计划

```
阶段一（4-5天）  后端 + 数据库
  ├── 建表 + 种子数据（菜单录入）
  ├── API：商品列表 / 提交订单 / 订单查询
  └── 本地调试完成

阶段二（4-5天）  小程序端
  ├── 菜单页面 + 购物车逻辑
  ├── 下单页面 + 订单列表
  └── 微信登录授权

阶段三（2天）    商家后台Web
  ├── 订单管理
  ├── 商品管理
  └── 店铺设置

阶段四（1-2天）  联调 + 部署 + 上线准备
  ├── 接口联调
  ├── 小程序审核准备
  └── 使用文档
```

**总计：约 11-14 天**
